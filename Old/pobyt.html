Problem Overview

My task was to improve the user experience for users to book nearby rooms(for hours) on-the-go. Below is a snippet of selected reviews from users that were upset with pobyt's services.


Existing design


Common user problems

1. App loads really slow - Performance issues
2. Confusing options - Not sure which one to select while selecting duration of stay
3. Price increase towards the end of flow - Users felt there was a sudden increase in price before payment

Business constraints

For business and marketing reasons, the team set out a “Pay per minute” campaign advertising that users can pay only a few cents for every minute they choose stay.
In reality, nobody is going to reserve a room for a few minutes. Yet, "Pay per Minute" was written into the motto of services offered by Pobyt

Problem Summary

1. User Expectations

The idea of "Pay Per Minute" sounds genuine and viable. This creates a lot of anticipation and expectations on Pobyt. But, the design clearly did not justify a clear user flow in navigating through its screens.

2. Unclear User flow

This was because users were unaware how to navigate through screens(especially with Hotel info screen).

3. Lack of Transparency

Pobyt calculates the tax component only in the later screens. This makes users feel "There is inconsistent pricing across pages"

4. Cognitive Overload

Pobyt tries to communicate a ton of information all the time. This increases the processing time thereby creating cognitive overload.

5. Inconsistency

Visuals and UX patterns are not consistent across the app. This could create a lot of confusion and sometimes frustration in users.

Design Goal

Having curated the list of problems that needed attention, I stepped up on a mission to solve the existing design problems that I thought were critical.

1. Improve the user flow for selecting duration of stay(hours and minutes)
2. Improve the overall visuals and information architecture of information displayed
3. Provide transparency in pricing
